# Campus Drop Backend

PostgreSQL 스키마는 Prisma로 정의되어 있습니다. 아래 ERD는 [campusdrop_server/prisma/schema.prisma](campusdrop_server/prisma/schema.prisma)를 기준으로 하며, **상세 필드·인덱스·제약은 해당 파일을 참고**하세요.

**유지보수**: `schema.prisma`를 변경한 경우 이 문서의 Mermaid 다이어그램도 함께 맞춰 주세요.

## ERD — 코어 도메인

```mermaid
erDiagram
  Identity {
    uuid id PK
  }
  Trait {
    uuid id PK
  }
  WeeklySurveySubmission {
    uuid id PK
    uuid identity_id FK
  }
  FriendWeeklySurveySubmission {
    uuid id PK
    uuid identity_id FK
  }
  SchoolProofSubmission {
    uuid id PK
    uuid identity_id FK
  }
  Cafe {
    uuid id PK
  }
  Matching {
    uuid id PK
    uuid user_a_id FK
    uuid user_b_id FK
    uuid cafe_id FK
  }
  MatchingFriendTalkRsvp {
    uuid matching_id PK
  }
  MatchingMeetingFeedback {
    uuid id PK
    uuid matching_id FK
    uuid identity_id FK
  }
  MeetingChatMessage {
    uuid id PK
    uuid matching_id FK
    uuid sender_id FK
  }

  Identity ||--o| Trait : "1:1 (shared PK)"
  Identity ||--o{ WeeklySurveySubmission : "romance weekly"
  Identity ||--o{ FriendWeeklySurveySubmission : "friend weekly"
  Identity ||--o{ SchoolProofSubmission : "school proof"

  Identity ||--o{ Matching : "as user_a"
  Identity ||--o{ Matching : "as user_b"
  Matching }o--o| Cafe : "optional cafe"
  Matching ||--o| MatchingFriendTalkRsvp : "max 1 per match"
  Matching ||--o{ MatchingMeetingFeedback : "feedback"
  Matching ||--o{ MeetingChatMessage : "chat"

  MatchingMeetingFeedback }|--|| Identity : "participant"
  MeetingChatMessage }|--|| Identity : "sender"
```

## ERD — 운영 · 분석 · 기타

`FriendTalkRsvpLink`, Analytics 테이블, `SchoolProofSubmission.reviewer_admin_id`는 Prisma에 `@relation`이 없거나 문자열 UUID만 두는 필드입니다. 아래 관계선은 **도메인상의 논리 참조**를 나타냅니다. `AccessLog`, `Admin`, `LandingLikeCounter`는 다른 테이블과 Prisma 관계가 없어 **엔티티만** 표시합니다.

```mermaid
erDiagram
  Identity {
    uuid id PK
  }
  Matching {
    uuid id PK
  }
  Admin {
    uuid id PK
  }
  AccessLog {
    uuid id PK
  }
  FriendTalkRsvpLink {
    string code PK
    uuid matching_id "optional logical"
    uuid identity_id "logical"
  }
  AnalyticsEvent {
    uuid id PK
    uuid session_id
    uuid user_uuid "logical Identity?"
  }
  AnalyticsSessionHeartbeat {
    uuid session_id PK
    uuid user_uuid "logical Identity?"
  }
  AnalyticsInteraction {
    uuid id PK
    uuid session_id
    uuid user_uuid "logical Identity?"
  }
  AnalyticsSessionUserLink {
    uuid session_id PK
    uuid user_uuid "logical Identity"
  }
  LandingLikeCounter {
    string key PK
  }
  SchoolProofSubmission {
    uuid id PK
    uuid reviewer_admin_id "logical Admin"
  }

  FriendTalkRsvpLink }o--|| Identity : "logical (no Prisma relation)"
  FriendTalkRsvpLink }o--o| Matching : "logical (nullable matching)"

  AnalyticsEvent }o--o| Identity : "logical user_uuid"
  AnalyticsSessionHeartbeat }o--o| Identity : "logical user_uuid"
  AnalyticsInteraction }o--o| Identity : "logical user_uuid"
  AnalyticsSessionUserLink }|--|| Identity : "logical user_uuid"

  SchoolProofSubmission }o--o| Admin : "logical reviewer_admin_id"
```
